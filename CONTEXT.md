# 이슈 템플릿 만들기

[우선순위] 간단한 설명

예시:
[A] 임베딩 대용량 처리 기능 추가
[B] 아이콘 클릭 시 활성화 표시
[C] API 문서 작성
[D] 성능 최적화 검토

## 우선순위
- [ ] A - 당장 해야 할 일 (긴급)
- [ ] B - 오늘 안에
- [ ] C - 이번 주 안에 (평일)
- [ ] D - 이번 달 안에

## 담당자
- Assignee: @username (1명만 지정)

## 이슈 설명
### 왜 이 이슈가 필요한가?
(이유 작성)

### 작업 내용
- [ ] 할 일 1
- [ ] 할 일 2

## 예상 일정
- 시작일: YYYY-MM-DD
- 완료 예정일: YYYY-MM-DD
  - A: 당일
  - B: 오늘 안(24시간 이내)
  - C: 이번 주 금요일까지
  - D: 이번 달 말일까지

## 추가 정보 (선택)
- 고객 시나리오: 
- 참고 자료: 
- 관련 이슈: #이슈번호

# Commit Convention 예시까지 생각해보기

feat: 기능 관련

docs: 문서 작업

fix: 버그 수정

ref: 기능은 동일하지만 내부처리 바꿀 때

test: 테스트관련 모든 작업

cicd: yaml 수정 작업

style:  주석이나 띄어쓰기 줄바꿈 등 

chore: 나머지 분류하기 어려운 기타작업들
